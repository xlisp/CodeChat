## 5. Checkpoint 状态

### 当前状态

```
checkpoints/codechat_2b/latest.pt  —  15GB
```

文件存在，大小合理（模型 5.3GB + Adam 状态 ~10GB = ~15GB）。

### 已知问题

1. **仅保留最新 checkpoint**：当前逻辑每 2,000 步保存，但始终覆盖 `latest.pt`。如果保存过程中断电或进程被杀，checkpoint 文件可能损坏，导致无法恢复。

2. **无历史 checkpoint**：无法回退到更早的训练状态（如 loss 最低点）进行对比实验。

### 建议改进

在 `scripts/base_train.py:78` 处增加带步数编号的备份：

```python
if step % args.save_every == 0 or step == args.max_steps:
    save_ckpt(ckpt_path, model, optim, step, cfg)
    step_path = ckpt_path.replace("latest.pt", f"step_{step}.pt")
    save_ckpt(step_path, model, optim, step, cfg)
    print(f"  saved -> {ckpt_path} + {step_path}")
```

注意：每个 checkpoint 15GB，每 2,000 步保存一份将额外占用约 **~105GB** 磁盘（剩余 7 次保存）。可改为每 5,000 步保存一份历史版本。
